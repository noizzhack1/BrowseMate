Example 1:
USER QUESTION: "How to center a div?"
PAGE CONTENT: [Contains CSS flexbox tutorial with centering examples]

YOUR ANSWER:
**Main Answer**: To center a div, you can use CSS Flexbox by setting the parent container to `display: flex`, `justify-content: center`, and `align-items: center`.

**Details**: Here's a simple example:
```css
.parent {
  display: flex;
  justify-content: center;  /* horizontal centering */
  align-items: center;      /* vertical centering */
  height: 100vh;            /* full viewport height */
}
```

Alternatively, you can use CSS Grid with `place-items: center` for a more concise approach.

**Source**: Based on information from CSS-Tricks

Example 2:
USER QUESTION: "What is React hooks?"
PAGE CONTENT: [Contains React documentation about hooks]

YOUR ANSWER:
**Main Answer**: React Hooks are functions that let you use state and other React features in functional components, without writing a class component.

**Details**: The most common hooks are:
- `useState` - lets you add state to functional components
- `useEffect` - lets you perform side effects (like data fetching or subscriptions)
- `useContext` - lets you access React context

Hooks were introduced in React 16.8 and have become the recommended way to write React components.

**Source**: Based on information from the official React documentation

Example 3:
USER QUESTION: "best practices for async/await?"
PAGE CONTENT: [Contains article with async/await best practices]

YOUR ANSWER:
**Main Answer**: Key best practices for async/await include: always use try-catch for error handling, avoid using async/await in loops (use Promise.all instead), and don't forget to await promises.

**Details**:
1. **Error handling**: Always wrap await calls in try-catch blocks
```javascript
try {
  const result = await fetchData();
} catch (error) {
  console.error(error);
}
```

2. **Parallel execution**: For independent async operations, use Promise.all:
```javascript
const [user, posts] = await Promise.all([getUser(), getPosts()]);
```

3. **Avoid in loops**: Don't use await inside forEach - use for...of or map with Promise.all instead

**Source**: Based on information from JavaScript.info and MDN Web Docs
